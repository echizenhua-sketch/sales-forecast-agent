## 1. 当前部署形态

本项目当前采用本地前后端分离运行：

| 模块 | 当前实现 | 端口 | 说明 |
|---|---|---:|---|
| 后端 API | FastAPI `main:app` | `8000` | 提供 `/api/...`、Swagger、文件上传、AI 对话 |
| 前端页面 | Vite 托管静态 HTML | `4173` | 默认进入 `login-v2.html`，不再使用 React 主入口 |
| 数据库 | MySQL 8 | `10.10.9.104:11123` | 库名 `supply_demand_forecast` |
| AI 网关 | OpenAI/Anthropic 兼容网关 | `10.10.9.104:8080` | 当前模型 `minimax-m3` |

访问入口：

```text

工作台:   http://127.0.0.1:4173/dashboard-v2.html
API:      http://127.0.0.1:8000/api/health
Swagger:  http://127.0.0.1:8000/docs
```

默认账号：


## 5. 数据库初始化和兼容列

### 5.1 初始化数据库

如果远端 MySQL 中还没有 `supply_demand_forecast` 库或基础表，执行：

```powershell
mysql -h 10.10.9.104 -P 11123 -u <user> -p < scripts/init_db_v2.2.sql
```

初始化完成后确认：

```powershell
python -c "from app.db import check_db_connection; print(check_db_connection())"
```

期望输出：

```text
True
```

### 5.2 启动时自动补齐兼容列

`main.py` 启动时会调用：

```python
app.scripts.ensure_mysql_schema.ensure_schema()
```

用于给既有 MySQL 表补齐当前静态前端和导出需要的 SKU 明细字段，例如：

- `business_unit`
- `product_series`
- `factory`
- `product_attribute`
- `product_category`
- `ex_factory_price`
- `launch_date`
- `satisfied_ka_before_25`

该脚本只做非破坏性加列，不删除旧字段。

## 9. 常见问题排查

### 9.1 登录页提示“网络错误”

不要只判断为后端没启动。按顺序检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/health
```

如果返回：

```json
{"status":"healthy","database":"disconnected"}
```

优先检查 MySQL：

- `10.10.9.104:11123` 是否可访问
- `supply_demand_forecast` 库是否存在
- `.env` 中 `DATABASE_URL` 是否正确
- MySQL 账号是否有建表、查询、插入权限

### 9.2 上传 Excel 报 MySQL 日期错误

曾出现过：

```text
Incorrect datetime value: 'NaT' for column 'launch_date'
```

处理原则：

- 不改数据库类型
- 不切换 SQLite
- 在业务服务输出层清洗 pandas `NaN/NaT`
- 当前修复位置：`app/services/forecast_service.py`

### 9.3 AI 助手报 `模型调用失败：'text'`

原因通常是模型响应不是 `content[0].text`：

- minimax-m3 可能先返回 `type=thinking`
- 真正回答在后续 `type=text`

当前修复位置：

```text
app/services/ai_service.py
```

排查时先抓响应 JSON 的顶层字段和 `content` 块类型，不要假设第一个块就是最终回答。

### 9.4 前端改动后没有生效

检查端口进程是否仍是旧服务：

```powershell
Get-NetTCPConnection -LocalPort 4173 -State Listen
Get-NetTCPConnection -LocalPort 8000 -State Listen
```

必要时关闭旧进程后重新启动：

```powershell
python -B -m uvicorn main:app --host 127.0.0.1 --port 8000
npm run dev --prefix web
```

### 9.5 页面跳到了风格不一致的独立页

当前静态页面应统一使用应用壳布局。新增菜单页时，应复用：

- 左侧固定菜单
- 顶部栏
- `bg-background`
- 右侧主内容区

不要为菜单项单独创建脱离左侧导航和顶部栏的简版页面。

## 10. 生产化注意事项

当前文档描述的是本地/内网部署形态。进入正式生产前，需要补齐：

- 反向代理和 HTTPS
- 正式密钥管理，不使用 `.env` 明文分发
- 数据库备份和恢复策略
- 日志轮转
- 进程守护，例如 Windows 服务、systemd、Supervisor 或容器编排
- 上传文件清理策略
- 模型网关限流、超时和错误监控

如果使用 Docker 部署到 `10.10.9.104`，必须遵守项目级部署约束：

- 所有运行数据、日志、配置、证书、临时文件放在项目源文件夹下
- Docker 网络必须显式指定 subnet
- 避开以下网段：
  - `172.28.0.0/16`
  - `172.22.0.0/16`
  - `192.168.0.0/16`
