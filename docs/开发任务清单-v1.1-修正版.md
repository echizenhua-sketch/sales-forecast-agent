def calculate_risk_level(total_supply, sar_total):
    if sar_total == 0:
        return ("NORMAL", "需求为0，无法计算风险", 0)
    
    gap = total_supply - sar_total
    gap_ratio = gap / sar_total
    gap_abs = abs(gap)
    
    if gap_ratio < -0.30 or (gap < 0 and gap_abs > 10000):
        return ("CRITICAL", f"缺口{gap:,.0f}（{gap_ratio:.1%}），严重缺货", 100 + abs(gap_ratio) * 100)
    elif -0.30 <= gap_ratio < -0.15 and gap_abs > 5000:
        return ("HIGH", f"缺口{gap:,.0f}（{gap_ratio:.1%}），可能缺货", 80 + abs(gap_ratio) * 100)

### 3.4.5 业务计算 - 可满足/未满足分配 🆕 **v1.1 新增**

**负责人**：后端  
**工期**：1.5 天  
**优先级**：P0  
**依赖**：3.3

**任务清单**：
- [ ] **实现可满足/未满足分配算法**：
  - 当缺口 < 0（供不应求）时：
    - 按各渠道 SAR 占比分配 total_supply
    - 计算各渠道的可满足数量
    - 计算各渠道的未满足数量 = SAR - 可满足
  - 当缺口 >= 0（供应充足）时：
    - 各渠道完全满足
    - 可满足 = SAR，未满足 = 0
- [ ] **计算 5 个渠道的可满足明细**：
  - `satisfied_province`, `satisfied_dealer`, `satisfied_ecommerce`, `satisfied_ka`, `satisfied_expansion`
- [ ] **计算 5 个渠道的未满足明细**：
  - `unsatisfied_province`, `unsatisfied_dealer`, `unsatisfied_ecommerce`, `unsatisfied_ka`, `unsatisfied_expansion`
- [ ] **验证计算公式**：
  - ✅ 可满足合计 = sum(5个渠道可满足)
  - ✅ 未满足合计 = sum(5个渠道未满足)
  - ✅ 可满足合计 + 未满足合计 = 合计SAR
- [ ] **编写单元测试**（覆盖供不应求和供应充足两种场景）

**验收标准**：
- 分配算法准确
- 验证公式成立
- 单元测试覆盖率 > 80%
- 边界条件（SAR=0）处理正确

**参考文档**：PRD v2.1 第 493 行（数据验证规则）

**算法伪代码**：
```python
def allocate_satisfied_unsatisfied(total_supply, sar_province, sar_dealer, 
                                   sar_ecommerce, sar_ka, sar_expansion):
    sar_total = sar_province + sar_dealer + sar_ecommerce + sar_ka + sar_expansion
    
    if sar_total == 0:
        return {
            'satisfied': [0, 0, 0, 0, 0],
            'unsatisfied': [0, 0, 0, 0, 0]
        }
    
    gap = total_supply - sar_total
    
    if gap < 0:  # 供不应求，按比例分配
        satisfied_province = (sar_province / sar_total) * total_supply
        unsatisfied_province = sar_province - satisfied_province
        # ... 其他 4 个渠道同理
    else:  # 供应充足，完全满足
        satisfied_province = sar_province
        unsatisfied_province = 0
        # ... 其他 4 个渠道同理
    
    return {
        'satisfied': [satisfied_province, satisfied_dealer, satisfied_ecommerce, 
                     satisfied_ka, satisfied_expansion],
        'unsatisfied': [unsatisfied_province, unsatisfied_dealer, unsatisfied_ecommerce,
                       unsatisfied_ka, unsatisfied_expansion]
    }
```
