## 3.3 风险等级判定标准（业务规则）

基于 PRD v2.1 第 2.6 节的建议，补充风险等级的具体判定逻辑：

### 3.3.1 风险等级定义

| 风险等级 | 英文标识 | 缺口比例判定条件 | 缺口绝对值判定条件 | 业务含义 | 建议动作 |
|---------|---------|----------------|------------------|---------|---------|
| 极高风险 | CRITICAL | 缺口 < -30% | 或缺口绝对值 > 10,000 | 严重缺货，影响订单交付 | 紧急排产 + 跨区调拨 |
| 高风险 | HIGH | -30% ≤ 缺口 < -15% | 且缺口绝对值 > 5,000 | 可能缺货，需提前准备 | 优先排产 + 密切监控 |
| 中风险 | MEDIUM | -15% ≤ 缺口 < -5% | - | 轻微缺货，需关注 | 正常排产 + 监控需求变化 |
| 正常 | NORMAL | -5% ≤ 缺口 ≤ +15% | - | 供需平衡 | 维持现有计划 |
| 压货风险 | OVERSTOCK | +15% < 缺口 ≤ +30% | - | 库存偏高，占用资金 | 适当减少排产 |
| 充足 | SUFFICIENT | 缺口 > +30% | - | 供应充足，无压力 | 可考虑削减排产 |

### 3.3.2 风险判定伪代码

```python
def calculate_risk_level(total_supply: float, sar_total: float) -> tuple[str, str, float]:
    """
    计算风险等级
    
    Args:
        total_supply: 总供应量
        sar_total: 合计SAR（总需求）
    
    Returns:
        (risk_level, risk_reason, risk_score)
        risk_score: 风险评分，用于排序（越高越危险）
    """
    if sar_total == 0:
        return ("NORMAL", "需求为0，无法计算风险", 0)
    
    gap = total_supply - sar_total
    gap_ratio = gap / sar_total  # 缺口比例
    gap_abs = abs(gap)  # 缺口绝对值
    
    # 极高风险：缺口比例 < -30% 或绝对缺口 > 10,000
    if gap_ratio < -0.30 or (gap < 0 and gap_abs > 10000):
        reason = f"缺口{gap:,.0f}（{gap_ratio:.1%}），严重缺货"
        if gap_abs > 10000:
            reason += f"，缺口绝对值{gap_abs:,.0f}超过阈值"
        return ("CRITICAL", reason, 100 + abs(gap_ratio) * 100)
    
    # 高风险：-30% ≤ 缺口比例 < -15% 且绝对缺口 > 5,000
    elif -0.30 <= gap_ratio < -0.15 and gap_abs > 5000:
        reason = f"缺口{gap:,.0f}（{gap_ratio:.1%}），可能缺货"
        return ("HIGH", reason, 80 + abs(gap_ratio) * 100)
    
    # 中风险：-15% ≤ 缺口比例 < -5%
    elif -0.15 <= gap_ratio < -0.05:
        reason = f"缺口{gap:,.0f}（{gap_ratio:.1%}），轻微缺货"
        return ("MEDIUM", reason, 50 + abs(gap_ratio) * 100)
    
    # 正常：-5% ≤ 缺口比例 ≤ +15%
    elif -0.05 <= gap_ratio <= 0.15:
        reason = f"供需平衡，缺口{gap:,.0f}（{gap_ratio:.1%}）"
        return ("NORMAL", reason, 10)
    
    # 压货风险：+15% < 缺口比例 ≤ +30%
    elif 0.15 < gap_ratio <= 0.30:
        reason = f"库存偏高，超额{gap:,.0f}（{gap_ratio:.1%}）"
        return ("OVERSTOCK", reason, 30)
    
    # 充足：缺口比例 > +30%
    else:
        reason = f"供应充足，超额{gap:,.0f}（{gap_ratio:.1%}）"
        return ("SUFFICIENT", reason, 20)
```

### 3.3.3 风险评分说明

`risk_score` 用于风险排序和优先级判断：
- **100+**：极高风险（最高优先级）
- **80-99**：高风险
- **50-79**：中风险
- **30-49**：压货风险
- **20-29**：充足（低优先级）
- **0-19**：正常（最低优先级）

风险列表默认按 `risk_score DESC` 排序，确保最危险的 SKU 优先展示。

### 3.3.4 待业务确认项

1. **缺口绝对值阈值**：
   - 极高风险：10,000（可根据业务规模调整）
   - 高风险：5,000（可根据业务规模调整）

2. **缺口比例阈值**：
   - 是否需要按产品线、事业部设置不同阈值？
   - 季节性产品是否需要特殊规则？

3. **风险评分权重**：
   - 当前方案：缺口比例越大，风险评分越高
   - 是否需要考虑 SKU 价值（如出厂价 × 缺口数量）？

**临时决策**：在业务方确认前，按上述标准实现，并在配置文件中预留阈值调整接口。

## 5. AI 能力实现方案（Python 版）

### 5.1 Python AI 服务架构

```python
# app/services/ai_service.py
from langchain.chat_models import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema import StrOutputParser
import re
import json

class AIService:
    """AI 问答服务"""
    
    def __init__(self):
        # 规则引擎
        self.rule_engine = RuleEngine()
        
        # LLM 客户端
        self.llm = ChatOpenAI(
            model="qwen-plus",  # 通义千问
            temperature=0,
            base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
        )
        
        # 问题路由器
        self.router = QuestionRouter()
    
    async def answer(
        self,
        question: str,
        task_id: int,
        session_id: str,
        db: AsyncSession
    ) -> dict:
        """回答问题"""
        
        # 1. 问题路由
        route_type = self.router.route(question)
        
        if route_type == "BOUNDARY_REJECT":
            return self._boundary_reject(question)
        
        # 2. 获取任务数据
        task_data = await self._get_task_data(task_id, db)
        
        if route_type == "RULE_ENGINE":
            # 规则引擎处理
            return await self.rule_engine.answer(question, task_data)
        
        # 3. LLM 处理
        return await self._llm_answer(question, task_data)
    
    def _boundary_reject(self, question: str) -> dict:
        """边界拒答"""
        return {
            "answer": "抱歉，当前系统仅支持基于上传数据的产销平衡分析，"
                     "无法回答关于外部市场预测、竞争对手分析等超出范围的问题。\n\n"
                     "我可以帮您：\n"
                     "• 查询 SKU 级缺口和风险\n"
                     "• 解释 SAR 组成和计算逻辑\n"
                     "• 提供补货和排产建议",
            "reference": None,
            "type": "boundary_reject"
        }
    
    async def _llm_answer(self, question: str, task_data: dict) -> dict:
        """LLM 回答"""
        
        # 构建 Prompt
        prompt = ChatPromptTemplate.from_messages([
            ("system", self._build_system_prompt()),
            ("human", "{question}\n\n当前数据：\n{data}")
        ])
        
        # 调用 LLM
        chain = prompt | self.llm | StrOutputParser()
        
        response = await chain.ainvoke({
            "question": question,
            "data": json.dumps(task_data, ensure_ascii=False, indent=2)
        })
        
        # 解析引用
        references = self._extract_references(response, task_data)
        
        return {
            "answer": response,
            "references": references,
            "type": "llm_generate"
        }


class RuleEngine:
    """规则引擎：处理固定模式问题"""
    
    async def answer(self, question: str, task_data: dict) -> dict:
        """规则匹配回答"""
        
        # 规则 1：查询特定 SKU 的缺口
        if match := re.search(r"SKU-?(\d+)", question, re.IGNORECASE):
            sku_code = f"SKU-{match.group(1)}"
            return await self._query_sku_gap(sku_code, task_data)
        
        # 规则 2：查询高风险 SKU
        if any(kw in question for kw in ["高风险", "缺货风险", "风险最高"]):
            return await self._query_high_risk(task_data)
        
        # 规则 3：查询总供应/总缺口
        if "总供应" in question or "总缺口" in question:
            return await self._query_summary(task_data)
        
        # 未匹配，返回 None
        return None
    
    async def _query_sku_gap(self, sku_code: str, task_data: dict) -> dict:
        """查询 SKU 缺口"""
        sku = task_data['skus'].get(sku_code)
        
        if not sku:
            return {
                "answer": f"未找到 {sku_code} 的数据，请检查 SKU 编码是否正确。",
                "reference": None,
                "type": "rule_engine"
            }
        
        answer = (
            f"**{sku['product_name']} ({sku_code})**\n\n"
            f"• 总供应：{sku['total_supply']:,.0f}\n"
            f"• 合计 SAR：{sku['sar_total']:,.0f}\n"
            f"• **供应缺口：{sku['gap']:+,.0f}**\n"
            f"• 风险等级：**{sku['risk_level_display']}**\n\n"
            f"风险原因：{sku['risk_reason']}"
        )
        
        return {
            "answer": answer,
            "reference": {
                "table": "库存风险明细清单",
                "sku": sku_code,
                "row": sku['row_index']
            },
            "type": "rule_engine"
        }
```

```python
# 后端分页 API
@router.get("/forecast/detail")
async def get_forecast_detail(
    task_id: int,
    page: int = 1,
    page_size: int = 100,  # 默认 100 条/页
    risk_level: str = None,
    db: AsyncSession = Depends(get_db)
):
    query = db.query(SKUForecastDetail).filter(
        SKUForecastDetail.task_id == task_id
    )
    
    # 风险筛选
    if risk_level:
        query = query.filter(SKUForecastDetail.risk_level == risk_level)
    
    # 按风险评分排序
    query = query.order_by(SKUForecastDetail.risk_score.desc())
    
    # 分页
    total = await query.count()
    items = await query.offset((page - 1) * page_size).limit(page_size).all()
    
    return {
        "total": total,
        "page": page,
        "page_size": page_size,
        "items": items
    }
```

**效果**：
- ✅ 每次仅传输 100 条数据，网络传输快
- ⚠️ 需要翻页操作，用户体验略差于虚拟滚动
