## 二、核心问题发现

### 问题1：SAR公式不一致（P0级别）

**PRD v2.0中的公式**：
```
5月合计SAR = 省大区SAR + 网络经销商SAR + 海外出口SAR + 内部调拨需求
```

**原始需求文档中的公式**：
```
5月合计SAR = 省大区SAR + 网络经销商SAR + 电商直营SAR + KA部SAR + 拓展部SAR
```

**Excel实际数据字段**：
- ✅ 有字段：`省大区SAR`、`网络经销商SAR`、`电商直营SAR`、`KA部SAR`、`拓展部SAR`
- ❌ 无字段：`海外出口SAR`、`内部调拨需求`

**影响范围**：
- 核心计算逻辑错误
- 数据模型设计错误
- 前端展示字段错误
- 可能导致整个系统的业务逻辑不正确

---

### 问题2：Excel字段清单缺失

**现状**：PRD 5.3节只提到"A列必须为SKU编码"、"B列必须为月度需求"，但未列出完整字段清单。

**实际情况**：Excel数据表包含67列字段，其中32个核心必需字段。

**影响**：文件解析模块开发缺少明确的字段规范。

## 五、影响评估

### 修正前的潜在风险

如果不修正SAR公式错误，将导致：

1. **核心算法错误**：系统计算出的SAR合计与实际业务不符
2. **数据字段缺失**：数据库缺少电商直营SAR、KA部SAR、拓展部SAR字段
3. **前端展示错误**：界面上显示错误的SAR组成项
4. **用户困惑**：用户上传的Excel有5个SAR字段，但系统只识别2个
5. **需求返工**：开发到一半发现业务逻辑错误，需大量返工

### 修正后的收益

1. ✅ **业务逻辑正确**：与原始需求和实际数据完全对齐
2. ✅ **开发规范明确**：32个字段清单为解析模块提供明确指引
3. ✅ **减少返工风险**：P0级别错误在开发前解决
4. ✅ **提升PRD质量**：综合评分从4.7/5提升至4.9/5
