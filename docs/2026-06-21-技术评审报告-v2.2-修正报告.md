# 技术评审报告 v2.2 修正报告

**文档**：产销预测智能工作台 - 技术评审报告（Python + MySQL 版）

## 一、修正背景

在对比技术评审报告与 PRD v2.1 后发现以下关键问题：

1. **P0 问题**：数据模型 SAR 字段与 PRD 不一致
   - 缺少第 5 个 SAR 字段（拓展部 SAR）
   - 字段命名错误（`sar_export` 应为 `sar_ecommerce`）

2. **P1 问题**：可满足/未满足明细缺失
   - PRD 要求 5 个渠道的独立统计
   - 当前只有汇总字段

3. **P1 问题**：风险等级判定标准未明确
   - PRD 已给出建议标准
   - 技术方案未落地实现逻辑

## 二、核心修正内容

### 修正 1：SAR 字段完整对齐（P0）

#### 修正前（v2.1）

```sql
-- SAR 需求组成（只有 4 个字段）
sar_province DECIMAL(18, 2) COMMENT '省大区SAR',
sar_dealer DECIMAL(18, 2) COMMENT '网络经销商SAR',
sar_export DECIMAL(18, 2) COMMENT '海外出口SAR',      -- ❌ 错误命名
sar_internal DECIMAL(18, 2) COMMENT '内部调拨需求',   -- ❌ 错误命名
sar_total DECIMAL(18, 2) COMMENT '合计SAR',
```

**问题**：
- ❌ 缺少第 5 个字段：拓展部 SAR
- ❌ `sar_export` 应为 `sar_ecommerce`（电商直营）
- ❌ `sar_internal` 应为 `sar_ka`（KA部）
- ❌ 与 PRD v2.1 第 493 行公式不符

#### 修正后（v2.2）

```sql
-- SAR 需求组成（5个独立渠道，与PRD v2.1保持一致）
sar_province DECIMAL(18, 2) COMMENT '省大区SAR',
sar_dealer DECIMAL(18, 2) COMMENT '网络经销商SAR',
sar_ecommerce DECIMAL(18, 2) COMMENT '电商直营SAR',    -- ✅ 修正命名
sar_ka DECIMAL(18, 2) COMMENT 'KA部SAR',               -- ✅ 修正命名
sar_expansion DECIMAL(18, 2) COMMENT '拓展部SAR',      -- ✅ 新增字段
sar_total DECIMAL(18, 2) COMMENT '合计SAR (= 省大区 + 网络经销商 + 电商直营 + KA部 + 拓展部)',
```

**影响范围**：
- 文件解析模块：需识别 5 个 SAR 字段
- 计算逻辑：SAR 合计公式调整
- 前端展示：SAR 组成细节需显示 5 项
- API 接口：响应字段调整

-- 验证迁移结果
SELECT
