## 2. 产品概述

### 2.1 产品名称

产销预测 Agent / 产销预测智能工作台

从界面文案看，这个产品同时强调两个定位：

- 作为“工作台”，承载数据上传、结果查看、异常分析、导出与日志
- 作为“Agent”，承载智能问答、解释说明、异常洞察和辅助决策

### 2.2 产品定位

`已确认事实`

- 登录页文案为“企业级产销智能决策工作台”
- 多个页面标题均为“产销预测 Agent”
- 设计系统说明明确指向“企业级供应链预测工具”
- 界面高频出现的业务关键词包括：SAR、供应缺口、排产、库存、预警、补货建议、Excel 上传、汇总结果、导出报表

`合理推断`

该产品是一个面向企业供应链/产销协同岗位的桌面端分析工作台，核心价值不是单纯生成预测结果，而是把“上传数据 -> 解析校验 -> 汇总计算 -> 风险识别 -> AI 问答解释 -> 导出决策依据”串成一条连续工作流。

### 2.3 产品目标

`合理推断`

从页面结构和文案看，本产品至少承担以下目标：

1. 将产销预测及供应表标准化导入系统
2. 自动解析上传文件并校验格式
3. 生成可用于月度产销平衡判断的汇总结果
4. 快速识别缺货风险、压货风险和待处理预警
5. 提供面向业务人员的 AI 辅助解读与追问入口
6. 为排产、库存和补货决策提供可导出、可解释的依据

### 2.4 目标用户与权限模型

`已确认事实`

### 2.5 核心使用场景

`合理推断`

1. 周期性上传月度预测/供应数据，形成新的分析任务
2. 文件解析失败时定位格式问题并重新上传
3. 查看月度供需缺口总览与高风险 SKU
4. 在汇总表中查看某个 SKU 的 SAR 组成与计算逻辑
5. 向 AI 助手追问”哪些 SKU 需要优先排产””SAR 差异原因”等问题
6. 导出汇总报表或单 SKU 逻辑明细，用于会议或审批

### 2.6 核心业务术语表

`待确认项（P0 优先级 - Kickoff 前必须确认）`

以下术语在界面中频繁出现，但完整定义需与业务方确认：

| 术语 | 初步理解 | 业务定义（待确认） | 计算公式（待确认） |
|------|---------|------------------|------------------|
| **SAR** | 销售与补货需求预测 | 【待确认】可能为 Sales And Replenishment，或 Supply Allocation Request | 已确认：省大区SAR + 网络经销商SAR + 电商直营SAR + KA部SAR + 拓展部SAR |
| **供应缺口（Gap）** | 供应量与需求量的差异 | 【待确认】正值表示供过于求，负值表示供不应求 | 已确认：5月总供应 - 5月合计SAR |
| **满足率（Service Level）** | 需求被满足的比例 | 【待确认】 | 推断：可满足合计 / 5月合计SAR |
| **库存周转天数** | 当前库存可支撑的销售天数 | 【待确认】 | 推断：期末库存 / 日均销售量 |
| **安全库存预留** | 为应对不确定性保留的最低库存 | 【待确认】 | 【待确认】固定比例？历史波动计算？ |
| **风险等级** | SKU 缺货或压货的风险程度 | 【待确认】见下表 | 【待确认】基于缺口比例或绝对值？ |

**风险等级划分标准（初步建议，待业务确认）**

| 风险等级 | 判定条件（建议） | 对应状态标签 | 业务含义 |
|---------|----------------|-------------|---------|
| 极高风险 | 缺口 < -30% 或缺口绝对值 > 10000 | 极高风险 | 严重缺货，需紧急排产 |
| 高风险 | -30% ≤ 缺口 < -15% | 缺货风险 | 可能缺货，需优先关注 |
| 中风险 | -15% ≤ 缺口 < -5% | 中风险 | 轻微缺货，需监控 |
| 正常 | -5% ≤ 缺口 ≤ +15% | 正常 | 供需平衡 |
| 压货风险 | 缺口 > +15% | 压货风险 | 库存积压，需调整排产 |
| 充足 | 缺口 > +30% | 充足 | 供应充足，无压力 |

`临时决策`

在业务方确认前，研发阶段暂按上述建议实现，预留配置化调整接口。

为企业用户提供安全、简洁的系统登录入口。

- 页面标题：`产销预测 Agent`

## 5.2 工作台 - 初始态

### 页面目标

在用户进入系统后，提供创建新预测任务的第一落点，并在无数据状态下引导完成上传。

### 已确认事实

- 页面主标题：`工作台`
- 说明文案：`开始新的产销预测任务`
- 明确步骤文案：`第一步：上传产销预测及供应表`
- 支持拖拽上传
- 支持按钮上传：`点击上传 Excel 文件`
- 支持格式：`.xlsx, .xls, .csv`
- 初始页显示 `支持 .xlsx, .xls, .csv 格式 (最大 20MB)`
- 最近处理记录区默认空态：`暂无处理记录，请先上传文件`
- KPI 卡片在初始态为空值：
  - 预测准确率 `--`
  - 库存周转天数 `--`
  - 待处理预警 `--`
- AI 助手在未上传文件前就已展示欢迎文案与建议能力
- 问答输入框在该状态下禁用，placeholder 为：`上传文件后可进行智能问答...`
- 推荐问题包括：
  - `如何优化库存准确度？`
  - `异常波动检测规则说明`

### 功能需求

1. 用户进入系统后可立即开始上传新任务文件
2. 页面需要清晰区分“无结果数据”与“可开始处理”
3. AI 助手可在无数据状态下提供能力说明，但不可执行正式分析问答
4. 页面需展示近期任务入口，便于回看历史处理结果

### 交互与状态要求

1. 当用户拖入文件时，上传区域应切换为拖拽高亮态
2. 当用户未上传任何文件时，指标区显示占位符而非错误
3. AI 输入区在文件上传前不可用
4. 最近处理记录在无历史时展示空态引导

### 合理推断

- 工作台初始态是整个系统的默认首页
- 初始页意图降低上手门槛，先引导上传，再逐步开放深度分析
- AI 助手在这里承担 onboarding 角色，不只是问答工具

### 待确认项

1. 上传区域是否支持多文件
2. 最近处理记录的保留周期和排序规则
3. 初始 KPI 占位值是否会改为读取上一次任务结果
4. 用户是否可以从历史记录直接恢复一个旧任务
5. **【P0 冲突】文件大小上限不一致**：
   - 工作台初始态显示"最大 20MB"
   - 文件处理页规范说明"请勿超过 50MB"
   - **临时决策**：统一为 **50MB**，并更新所有页面文案
   - **待设计确认**：最终标准以哪个为准

### 验收标准

- [ ] 用户登录后默认进入工作台初始态
- [ ] 显示"开始新的产销预测任务"引导文案
- [ ] 上传区支持拖拽和点击上传，支持 .xlsx / .xls / .csv 格式
- [ ] 文件大小超过 50MB 时显示错误提示（临时统一为 50MB）
- [ ] 拖入文件时上传区高亮显示
- [ ] KPI 卡片在无数据时显示 `--` 占位符，不显示错误状态
- [ ] AI 输入框在无文件时禁用，显示"上传文件后可进行智能问答..."
- [ ] 最近处理记录区在无历史时显示"暂无处理记录，请先上传文件"
- [ ] AI 推荐问题可见但点击时提示需先上传文件

在文件上传后，向用户展示解析进度、处理步骤、错误原因、操作建议和处理历史。

### 已确认事实

- 页面标题：`上传预测明细表`
- 当前版本文案：`Enterprise v4.2.1`
- 展示已上传文件名，例如：`2024Q2_产销预测_v1.xlsx`
- 展示文件元信息：`文件大小：2.4MB | 修改日期：2024-05-20 14:30`
- 页面展示进度状态：`文件解析中 (65%)...`
- 页面展示预计剩余时间：`预计还需 12 秒`
- 步骤条包含 4 步：
  - 文件上传
  - 数据解析
  - 汇总计算
  - 结果存库
- 页面展示错误状态：`解析失败`

  - `重新上传`
  - `查看规范手册`

`待确认项（P1 优先级 - 技术评审前确认）`

详细的 AI 能力范围和技术约束已在 5.5 节中说明，此处补充实现层面的待确认项：

1. **AI 模型选择**：
   - 自研模型 vs 集成第三方 API（如 GPT-4、通义千问、文心一言）
   - 如使用第三方 API，成本和延迟是否可接受
   - 是否需要针对供应链领域进行微调

2. **数据检索机制**：
   - 使用向量数据库（如 Milvus、Pinecone）进行语义检索
   - 使用传统 SQL 查询 + 关键词匹配
   - 混合检索（向量 + 关键词）

3. **上下文管理**：
   - 多轮对话上下文如何存储（内存 / Redis / 数据库）
   - 会话超时时间（建议 30 分钟）
   - 上下文窗口大小（建议保留最近 10 轮）

4. **引用准确性保证**：
   - 如何确保 AI 引用的表格位置准确无误
   - 引用格式："表: 库存风险明细清单 (Row 1, SKU-002)"
   - 引用是否支持点击跳转到对应数据行

5. **快捷动作实现**：
   - "建议排产计划"是否触发独立页面 / 弹窗 / 右侧面板
   - "查看库存预警"是否跳转到预警列表页
   - 快捷动作是否需要二次确认

`临时决策`

在 AI 技术方案确认前，MVP 阶段可采用以下简化方案：
- 使用规则引擎 + 模板匹配处理常见问题（如"哪些 SKU 缺货"）

- 引用机制先实现文本标注，点击跳转功能后续迭代
